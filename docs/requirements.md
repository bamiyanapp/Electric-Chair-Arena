# 要件定義書 (Electric Chair Arena)

## 1. プロジェクト概要
「電気イスゲーム」を再現し、人間プレイヤーが多様なAIモデルから対戦相手を選択して遊べるシステム。
人間とAIとの対戦ログ・戦績を蓄積し、戦績データを通じてAIモデルが継続的に学習を行い、段階的により強力な戦術を身につけ成長していくことを目指す。

## 2. ターゲット
- 電気イスゲームのファン
- 心理戦や駆け引きが好きなゲームプレイヤー
- 人間のプレイスタイルに適応し、学習によって強くなるAIとの対戦に興味があるユーザー・開発者

## 3. ゲームルール要件

### 3.1 イスの配置と構成
- **イスの数**: イスは1から12まで配置されている。
- **配置形状**: 時計の文字盤と同じように、1〜12が時計回りの円状に並んでいる。

### 3.2 勝利・敗北条件
- **勝利条件**: 先に **40点先取** したプレイヤーが勝利。
- **敗北条件**: 合計で **3回電気を喰らう**（感電する）と敗北が決定する。
- **電気を喰らった際の影響**: 電気（電流）を喰らうと、そのプレイヤーのそれまでのスコアは **0点にリセット** される。

### 3.3 ゲームの進行
- 人間プレイヤーとAIモデルが交互に「座るイスの選択」と「電流の設置」を交互に行う。
- イスには、電流を設置する側のプレイヤーによって事前に一部に電流が仕掛けられる。
- **ゲーム終了のタイミング**:
  - いずれかのプレイヤーが40点に到達した場合
  - いずれかのプレイヤーが3回感電した場合
  - **最後に椅子が1つになった場合**（残る椅子が1つになった場合は、ゲーム終了とする）

## 4. システム要件

### 4.1 フロントエンド
- **対戦画面 (ゲームプレイ画面)**:
  - 円形（時計状）に配置された1〜12のイスの視覚的UI表現
  - 人間プレイヤーが直感的に椅子を選択（Click）または電流を設置できるUI
  - スコアと感電回数（ライフ）、および残り椅子の表示
  - 現在のターン（先攻/後攻、設置/選択）の明示
- **AI選択ロビー画面**:
  - 特徴の異なるAIモデル（個性付きAI3種「岡野陽一風」「小籔千豊風」「千原ジュニア風」、ランダムAI、期待値計算AI、ナッシュ均衡AI）から対戦相手を選択できる画面
  - 各AIモデルのステータス、戦績、現在のレーティングを表示
- **人間対人間（ローカルPvP）画面**:
  - 同一画面上で2人の人間プレイヤーが交互に操作して対戦するモード
- **リーダーボード画面**:
  - 人間との戦績を通じたAIモデルのELOレーティング、勝利数のランキング
- **過去のスコアボード一覧画面**:
  - 保存済みの対戦結果（スコア・勝敗）を一覧表示する画面

### 4.2 バックエンド
- **ゲームエンジン**:
  - ルールに基づくゲーム状態遷移（イスの選択、電流設置の登録、感電判定、スコア加算、勝敗・終了判定）
- **AI意思決定ロジック**:
  - 個性付きAI3種はキャラクター性を反映したヒューリスティック分岐、期待値計算AIは得点効率に基づく選択、ナッシュ均衡AIは混合戦略の計算により行動を決定する。いずれもLLMは使用しない。
  - 各AIの行動には、寸劇調の理由付きセリフ（reasoning）が付随する。
- **対戦実況生成**:
  - 対戦終了後、Gemini APIを呼び出して実況コメントを生成する（`GEMINI_API` 未設定時はモック文言）。
- **レーティング更新**:
  - 対戦終了時に勝敗結果からELOレーティングを再計算し、AIモデルの `rating` / `matchCount` を更新する。AIの行動パラメーター自体（確率テーブル等）は更新されず、対戦を重ねても行動ロジックは変化しない。

### 4.3 データベース
- **AI Models (Players)**: AIモデルの定義、戦績・レーティング情報の保持（インメモリ、永続化なし。詳細は `docs/db_design.md` 参照）
- **Matches**: 人間とAI、または人間同士の各マッチの勝敗結果、および詳細なターン履歴ログの保存（インメモリ、永続化なし）
