# 要件定義書 (Electric Chair Arena)

## 1. プロジェクト概要
「電気イスゲーム」を再現し、人間プレイヤーが多様なAIモデルから対戦相手を選択して遊べるシステム。
人間とAIとの対戦ログ・戦績を蓄積し、戦績データを通じてAIモデルが継続的に学習を行い、段階的により強力な戦術を身につけ成長していくことを目指す。

## 2. ターゲット
- 電気イスゲームのファン
- 心理戦や駆け引きが好きなゲームプレイヤー
- 人間のプレイスタイルに適応し、学習によって強くなるAIとの対戦に興味があるユーザー・開発者

## 3. ゲームルール要件

### 3.1 イスの配置と構成
- **イスの数**: イスは1から12まで配置されている。
- **配置形状**: 時計の文字盤と同じように、1〜12が時計回りの円状に並んでいる。

### 3.2 勝利・敗北条件
- **勝利条件**: 先に **40点先取** したプレイヤーが勝利。
- **敗北条件**: 合計で **3回電気を喰らう**（感電する）と敗北が決定する。

### 3.3 ゲームの進行
- 人間プレイヤーとAIモデルが交互に「座るイスの選択」と「電流の設置」を交互に行う。
- イスには、電流を設置する側のプレイヤーによって事前に一部に電流が仕掛けられる。
- **ゲーム終了のタイミング**:
  - いずれかのプレイヤーが40点に到達した場合
  - いずれかのプレイヤーが3回感電した場合
  - **最後に椅子が1つになった場合**（残る椅子が1つになった場合は、ゲーム終了とする）

## 4. システム要件

### 4.1 フロントエンド
- **対戦画面 (ゲームプレイ画面)**:
  - 円形（時計状）に配置された1〜12のイスの視覚的UI表現
  - 人間プレイヤーが直感的に椅子を選択（Click）または電流を設置できるUI
  - スコアと感電回数（ライフ）、および残り椅子の表示
  - 現在のターン（先攻/後攻、設置/選択）の明示
- **AI選択ロビー画面**:
  - 特徴の異なるAIモデル（例：ランダム、慎重派、アグレッシブ、学習型）から対戦相手を選択できる画面
  - 各AIモデルのステータス、戦績、現在のレーティングを表示
- **リーダーボード / 成長履歴画面**:
  - 人間との戦績を通じたAIモデルのELOレーティング、勝率のランキング
  - AIがどれだけ学習して強くなったかを示す成長グラフや統計情報の表示

### 4.2 バックエンド
- **ゲームエンジン**:
  - ルールに基づくゲーム状態遷移（イスの選択、電流設置の登録、感電判定、スコア加算、勝敗・終了判定）
- **学習・成長システム (AI Feedback Loop)**:
  - 対戦終了時に勝敗および詳細な対戦ログ（どの状況でどの椅子を選んだか、電流を避けたか等）を解析
  - 対戦結果をインプットとして、AIの行動パラメーター（確率テーブル、ヒューリスティック重み、またはLLMのプロンプト/微調整データ）を自動アップデートし、戦うたびに強化される仕組みを提供

### 4.3 データベース
- **AI Models (Players) テーブル**: AIモデルの定義、学習用パラメーター、および戦績・レーティング情報の保持
- **Matches テーブル**: 人間とAIの各マッチの勝敗結果、および詳細なターン履歴ログ（JSON）の保存（AIの再学習・傾向分析用の教師データとして活用）
