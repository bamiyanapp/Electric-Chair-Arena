# 要件定義書 (Electric Chair Arena)

## 1. プロジェクト概要
「電気イスゲーム」を再現し、AIプレイヤーの対戦、学習、および評価を行うシステム。
ユーザー（人間）とAIプレイヤー、またはAIプレイヤー同士での大量対戦のシミュレーションと勝率、レーティング評価を行う。

## 2. ターゲット
- 電気イスゲームのファン
- AIの行動パターン、強化学習、LLM（GPT/Claude/Gemini）による対戦に興味がある開発者

## 3. ゲームルール要件

### 3.1 イスの配置と構成
- **イスの数**: イスは1から12まで配置されている。
- **配置形状**: 時計の文字盤と同じように、1〜12が時計回りの円状に並んでいる。

### 3.2 勝利・敗北条件
- **勝利条件**: 先に **40点先取** したプレイヤーが勝利。
- **敗北条件**: 合計で **3回電気を喰らう**（感電する）と敗北が決定する。

### 3.3 ゲームの進行
- プレイヤーは交互に座るイスを選択する。
- イスには事前に一部に電流が仕掛けられる。
- **ゲーム終了のタイミング**:
  - いずれかのプレイヤーが40点に到達した場合
  - いずれかのプレイヤーが3回感電した場合
  - **最後に椅子が1つになった場合**（残る椅子が1つになった場合は、ゲーム終了とする）

## 4. システム要件

### 4.1 フロントエンド
- **対戦画面**:
  - 円形（時計状）に配置された1〜12のイスの視覚的UI表現
  - 各イスの選択、感電状態、および残り椅子の表示
  - スコアと感電回数（ライフ）の表示
- **マッチシミュレーター画面**: AIプレイヤーを選択して対戦を開始できる画面
- **ランキング画面**: ELOレーティングに基づいたリーダーボード表示

### 4.2 バックエンド
- **ゲームエンジン**:
  - ルールに基づくゲーム状態遷移（イスの選択、感電判定、スコア加算、勝敗・終了判定）
- **対戦シミュレータ**: AIプレイヤー同士を連続して対戦させ、ログを保存する機能

### 4.3 データベース
- **Players テーブル**: AIの登録、パラメータ、および戦績・レーティング情報の保持
- **Matches テーブル**: 各マッチの勝敗結果、および詳細なターン履歴ログ（JSON）の保存
