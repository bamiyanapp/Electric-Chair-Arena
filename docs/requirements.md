# 要件定義書

## 1. プロジェクト概要
日本企業に存在する「形式主義」「ハンコ文化」「マナー文化」を風刺したジョークWebアプリ。

## 2. ターゲット
- 一次ターゲット：IT企業新人
- 二次ターゲット：SIer勤務者、大企業勤務者、稟議文化経験者

## 3. 機能要件

### 3.1 捺印システム
- 申請書への捺印機能（ドラッグ＆ドロップ、回転、配置）
- 判定ロジック：
  - 印影の角度（上司に向かってお辞儀をするように傾ける「お辞儀ハンコ」等）
  - 印影の濃さ、鮮明さ
  - 他の印影との距離、位置関係
  - ステージごとの独自ルール

### 3.2 ステージ構成
1. **課長承認**: 基本的な捺印作法（角度、濃さ）
2. **部長承認**: 組織内での敬意の表現（位置関係）
3. **本部長承認**: 高度なマナー（黄金比、会釈角度）
4. **マナー講師**: 精神論的な評価（誠意、焦り）
5. **ラスボス（AIマナー協会会長）**: 極限のマナー

### 3.3 フィードバック
- 差し戻しメッセージ（理由の提示）
- AIによる講評生成（将来機能：OpenAI/Gemini API活用）

## 4. 非機能要件
- 直感的なUI/UX（ゲーミフィケーション要素）
- 拡張性：新しいマナー（ステージ）を容易に追加できる構成

## 5. 完了条件（エンディング）
- 全ステージクリア後、ハンコマスターに認定される
- 直後に「電子承認システム導入」の通知により全ての努力が無に帰す演出
